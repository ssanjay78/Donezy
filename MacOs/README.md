# Donezy (iOS)

The iOS port of the Android **Donezy** habit & routine tracker — *Plan it · Do it · Done it.*
This mirrors the Android app's UI and backend behaviour for iPhone, built with SwiftUI,
a raw-SQLite backend, local notifications, and a WidgetKit home-screen widget.

It is a faithful re-implementation of `../HobbyLog` (Kotlin / Jetpack Compose). Same data
model, same SQLite schema and migrations, same categories / templates / quick-log presets,
same achievements / streaks / weekly goals, same theming, and the same screens.

## Requirements

- **Xcode 16** or newer (iOS 17 deployment target — the widget uses interactive
  `AppIntents`, which require iOS 17+).
- An Apple Developer team for code signing (free personal team is fine for device runs;
  the Simulator needs none).

## Open & run

```bash
open Donezy.xcodeproj
```

Then in Xcode:

1. Select the **Donezy** scheme and a simulator (e.g. iPhone 15).
2. Press ⌘R.

The project, both targets (app + widget extension), the shared scheme, and the App Group
entitlement are all pre-configured. The `project.pbxproj` is generated by
[`gen_pbxproj.py`](gen_pbxproj.py) — re-run `python3 gen_pbxproj.py` if you add/rename
source files, then reopen the project.

### Signing / App Group

The app and widget share their SQLite database, photos, and preferences through the App
Group `group.com.swarnkary.donezy`. On first build, select your team under
**Signing & Capabilities** for *both* targets. If the App Group can't be provisioned, the
app still runs — it falls back to its own Documents directory (see
[`Shared/AppGroup.swift`](Shared/AppGroup.swift)) — but the widget won't see the same data
until the group is enabled.

To use your own bundle identifier, edit `APP_BUNDLE_ID` / `WIDGET_BUNDLE_ID` (and
optionally `DEV_TEAM`) in `gen_pbxproj.py` and the group id in `Shared/AppGroup.swift`
plus the two `*.entitlements` files, then regenerate.

## Feature parity with the Android app

| Area | Android | iOS |
|------|---------|-----|
| Storage | SQLite via `SQLiteOpenHelper`, `hobby_log.db` v4 | SQLite via `SQLite3` C lib, same file & schema & migrations |
| Backup format | JSON v4 | Same JSON (round-trips between platforms) |
| Reminders | AlarmManager + BroadcastReceiver re-arm | `UNUserNotificationCenter`; recurrences re-armed on foreground & on Done/Snooze |
| Recurrence | none / hourly / daily / weekly / monthly | identical rules & math |
| Notification actions | Done / Snooze (picker) / Dismiss | Done / Snooze 1h / Dismiss |
| Streak rescue | 23:30 alarm | 23:30 repeating local notification, re-evaluated on launch |
| Home-screen widget | `AppWidgetProvider` 2×1 next-due | WidgetKit small/medium, interactive quick-log |
| Theme | Material3 light/dark/system | matching palette, light/dark/system |
| Photos | gallery + camera | Photos picker (camera via picker) |
| Streaks / achievements / weekly goals / insights chart | ✓ | ✓ |

## Platform differences worth knowing

These are inherent iOS/Android differences, kept as close as the platform allows:

- **Reminders.** iOS has no AlarmManager-style broadcast that runs code when an alarm
  fires. Donezy schedules one local notification per tracker and *re-arms recurrences when
  the app foregrounds* (`NotificationManager.rearmAll`) or when you tap **Done**/**Snooze**
  on the notification. A recurring reminder therefore advances reliably as long as you open
  the app or action the notification — which matches normal usage.
- **Snooze.** Android shows a custom multi-option snooze picker from the shade. iOS
  notification action buttons can't host a custom multi-choice UI, so the shade offers a
  single **Snooze 1h**; the in-app reminder card still gives full date/time/recurrence
  control.
- **Widget quick-log.** Uses an interactive `AppIntent` (iOS 17+) writing to the shared
  database, then asks all timelines to reload — the analogue of the Android widget's
  one-tap "Logged".

## Notes

- All data is local: no backend, no analytics, no network calls — same as Android.
- The app icon and accent are placeholder asset-catalog entries; drop in a 1024×1024 icon
  under `Donezy/Assets.xcassets/AppIcon.appiconset` before shipping.
